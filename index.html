Crux is a statically-typed language, that compiles to JavaScript, with an emphasis on familiarity, productivity, and soundness.

Main Ideas

Crux is all about small ideas that smoothly fit together without clumsy seams or corner cases. Crux is 100% type safe, but thanks to bidirectional type inference, you almost never have to spell out the names of your types.

Crux code integrates directly with existing JavaScript -- you can build Crux libraries for publishing on npm or use existing JavaScript libraries. Unlike other strongly-typed languages that compile to JavaScript (e.g. Scala.js, GHCJS, Emscripten), Crux's concepts map directly to ES6.

(The following claim is yet to be proven, haha!) In fact, Crux code is often faster than equivalent JavaScript because methods in Crux are statically dispatched rather than dynamically looked up from an object. This means you can write your code naturally and know that the optimizer will inline short methods.

Crux is sound - never worry about missing properties or "undefined is not a function" again.

Unlike TypeScript, Crux allows custom data types to use the same syntax and operators as native data types, such as tuples, 64-bit integers, sets, maps, slices, and ranges. The length of a value is always accessed with `len(x)` and iteration across any data structure uses the same for loop structure.
