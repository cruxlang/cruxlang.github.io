Main Ideas

Bidirectional Type Inference

Unlike other statically typed languages, Crux's type inference algorithm means the programmer is never forced to write type annotations on functions or variables.

{% highlight crux %} let output = mutable [] output->append("hello world") // output is inferred to be a MutableArray {% endhighlight %}

Integration with npm

Crux code integrates directly with existing JavaScript -- either export your code as an npm package or write a program that leverages existing JavaScript libraries.

Unlike other compile-to-JavaScript languages (e.g. Scala.js, GHCJS, Emscripten), Crux's concepts map directly to ES6, so the generated code is clean, debuggable, and has predictable performance.

Soundness

Crux is sound - never worry about missing properties or "undefined is not a function" again. This is especially helpful when refactoring.

Rich Data Types

Unlike TypeScript, Crux allows custom data types to use the same syntax and operators as native data types, such as tuples, 64-bit integers, sets, maps, slices, and ranges. The length of a value is always accessed with `len(x)` and iteration across any data structure uses the same for loop structure.
