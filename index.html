Crux is a statically-typed language, compiling to JavaScript, with an emphasis on familiarity, productivity, and soundness.

Main Ideas

Crux is all about small ideas that smoothly fit together without clumsy seams or weird corner cases. Crux is 100% type safe, but thanks to bidirectional type inference, you almost never have to spell out the names of your types.

Crux code integrates directly with existing JavaScript -- you can build Crux libraries for publishing on npm or use existing JavaScript libraries.

Crux is sound [TODO: link to soundness page] - never worry about missing properties or "undefined is not a function" again.

Crux's syntax is designed to be familiar to anyone coming from JavaScript or Python.

Unlike other strongly-typed languages that compile to JavaScript, like Scala.js or GHCJS or Emscripten, Crux maps directly to ES6 semantics. This means that code written in Crux is as fast as JavaScript or TypeScript.

In fact, Crux code is often faster [benchmarks needed] than similar JavaScript, because methods in Crux are statically-dispatched rather than dynamically looked up from an object. This means you can write your code naturally and know that the optimizer will inline short methods.

Crux has tuples.

Iteration uses exactly the same syntax no matter the data structure: arrays, immutable arrays, sets, maps, slices, ranges.
